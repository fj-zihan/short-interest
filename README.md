# Short Interest Factor Pipeline

This repository implements a cross-sectional factor pipeline using Dagster for orchestration and S3 for data storage. The system processes S&P 500 constituents to generate a short interest-based cross-sectional signal.

---

## Pipeline

```
sp500_universe
      ↓
short_interest_raw
      ↓
short_interest_factor
```

The pipeline supports:
- Full backfill computation
- Incremental updates for latest data

## Components

### 1. Universe

- Uses S&P 500 constituents as the current universe
- Implemented as a Dagster asset (`sp500_universe`)
- Reads from `data/static/SP500_universe.csv`, generated by `scripts/build_universe.py`
- Planned upgrade: point-in-time universe

### 2. Raw Data Asset

**Asset:** `short_interest_raw`

- Fetches short interest data from external provider (Massive API)
- Standardizes raw dataset for downstream use
- Input to factor computation

### 3. Factor Asset

**Asset:** `short_interest_factor`

- Cross-sectional z-score per date
- Winsorization before normalization
- Outputs standardized factor values

### 4. Full / Incremental Assets

- `short_interest_factor_full` — full historical backfill
- `short_interest_factor_incremental` — latest data only

### 5. Infrastructure Layer

#### IO Manager

Writes all assets to S3 with lineage-aware structure:

```
model=us_fundamental/
  dataset={asset_name}/
    lineage=live/
    lineage=backfill/
      build={timestamp}/
```

#### Data Provider

- `MassiveProvider` implements a short interest data interface
- Encapsulates external API logic
- Can be swapped without modifying asset code

## Setup

### 1. Clone and create a virtual environment

```bash
git clone <repo-url>
cd files2
python3 -m venv venv
source venv/bin/activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Generate the universe file

```bash
python3 scripts/build_universe.py
```

This fetches the current S&P 500 constituent list from Wikipedia and writes it to `data/static/SP500_universe.csv`. Run once before materializing the pipeline.

### 4. Configure environment variables

Create a `.env` file in the project root:

```
MASSIVE_API_KEY=your_api_key_here
S3_BUCKET=your_bucket_name
```

AWS credentials should be configured via `~/.aws/credentials` or the standard environment variables (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`).

## How to Run

### Start Dagster UI

```bash
dagster dev -m risk_models.us_fundamental.short_interest
```

### Run incremental job

```bash
dagster job execute -m risk_models.us_fundamental.short_interest \
  -j short_interest_incremental_job
```

### Run full backfill job

```bash
dagster job execute -m risk_models.us_fundamental.short_interest \
  -j short_interest_full_backfill_job
```

### Run tests

```bash
pytest tests/
```

## Design Principles

- Separation of raw data and factor computation
- Deterministic cross-sectional factor generation
- Support for full + incremental workflows
- S3 as persistent storage layer
- Modular provider abstraction

## Notes

- Current universe is a static S&P 500 snapshot (non-point-in-time)
- Factor is a cross-sectional standardized signal (winsorized z-score)
- System is structured for future point-in-time universe upgrade
